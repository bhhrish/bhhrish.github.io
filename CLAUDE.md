## Project Overview

"Unmasking Dharma" — a personal Hugo blog at [bhhrish.github.io](https://bhhrish.github.io) focused on informal exploration of the Ramayana and Mahabharata: clarifying misunderstood concepts, busting myths, and inviting counter-opinions. Tone is personal and open-minded, not authoritative.

## Hugo Commands

```bash
# Serve locally with live reload (includes drafts)
hugo server -D

# Build for production
hugo --minify

# Create a new post
hugo new posts/<slug>.md
```

`public/` is a build artifact — it is gitignored and generated by CI.

## Architecture

**Theme**: PaperMod, installed as a git submodule at `themes/PaperMod/`. Do not edit theme files directly — override via `layouts/` or `assets/` at the repo root instead.

**Deployment**: GitHub Actions (`.github/workflows/deploy.yml`) builds on every push to `main` and publishes via the GitHub Pages API. The workflow uses `submodules: recursive` to pull PaperMod. To enable deployment: go to repo Settings → Pages → Source → set to "GitHub Actions".

**Content structure**:
```
content/
  about.md          # Static about page
  posts/            # All blog posts live here
```

**Front matter fields in use**: `title`, `date`, `draft`, `tags`, `description`. Tags drive navigation — use `ramayana`, `mahabharata`, `philosophy`, `dharma`, `characters`, `myth-busting` as appropriate.

**Menu**: Configured in `hugo.toml` under `[menu]`. The Ramayana and Mahabharata nav links point to their tag archive pages (`/tags/ramayana/`, `/tags/mahabharata/`) — so tagging posts correctly is what populates those sections.

**Deprecation warnings at build time**: Two warnings (`Language.LanguageCode`, `Language.LanguageDirection`) come from inside PaperMod's templates, not our config. They are harmless and will clear when PaperMod updates.
